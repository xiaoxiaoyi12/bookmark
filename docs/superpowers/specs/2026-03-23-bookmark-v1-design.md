# BookMark V1 设计文档

## 概述

BookMark 是一个 Web 端电子书阅读 + 笔记工具。左侧阅读电子书，右侧做笔记，选中文字可自动引用到笔记。V1 为纯本地应用，无需后端。

## 目标用户

有电子书阅读习惯、希望在阅读过程中高效记录和整理笔记的用户。

## 核心功能

### 1. 书库管理

- 本地导入电子书文件（拖拽或文件选择器）
- 书库列表展示（封面、书名、作者、阅读进度）
- 支持删除已导入的书籍
- 书籍文件存储在 IndexedDB 中

### 2. 电子书阅读器

- **EPUB 渲染**：基于 epub.js，支持目录导航、翻页、字体大小调节
- **PDF 渲染**：基于 PDF.js，支持缩放、页码跳转
- **TXT 渲染**（P1）：编码自动检测，章节正则匹配生成目录
- 记住阅读进度，下次打开自动定位

### 3. 文字选中与高亮

- 选中文字后弹出工具栏：高亮、添加到笔记、复制
- 支持多种高亮颜色
- 高亮持久化存储，重新打开书籍后高亮仍在
- 点击已有高亮可查看/跳转到对应笔记

### 4. 笔记编辑器

- 富文本编辑器（所见即所得），基于 Tiptap
- 支持基础格式：标题、粗体、斜体、列表、引用块、代码块
- 选中书中文字后自动插入引用块到笔记（包含原文 + 位置信息）
- 每本书自动创建一个专属笔记

### 5. 数据存储

- 所有数据存储在浏览器 IndexedDB 中
- 数据模型：
  - **books**：书籍元数据（标题、作者、封面、格式、文件数据）
  - **highlights**：高亮记录（书籍ID、位置信息、颜色、文本内容）
  - **notes**：笔记内容（书籍ID、富文本JSON）
  - **reading_progress**：阅读进度（书籍ID、位置）
- 支持导出笔记为 Markdown 文件

## 技术方案

### 技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| 框架 | React 18 + TypeScript | 生态成熟，组件化开发 |
| 构建 | Vite | 快速，开箱即用 |
| EPUB 渲染 | epub.js | Web 端最成熟的 EPUB 方案 |
| PDF 渲染 | PDF.js (pdfjs-dist) | Mozilla 维护，事实标准 |
| 富文本编辑器 | Tiptap | 基于 ProseMirror，可扩展性强，文档好 |
| 本地存储 | Dexie.js (IndexedDB 封装) | API 友好，支持索引查询 |
| 样式 | Tailwind CSS | 快速开发，与 React 配合好 |
| 状态管理 | Zustand | 轻量，API 简洁 |

### 页面结构

```
App
├── 书库页 (BookLibrary)
│   ├── 导入按钮 / 拖拽区域
│   └── 书籍网格列表 (BookCard)
│
└── 阅读页 (ReaderPage)
    ├── 左侧面板：目录 (TableOfContents)
    ├── 中间：阅读器 (ReaderView)
    │   ├── EpubReader
    │   └── PdfReader
    ├── 右侧面板：笔记 (NotePanel)
    │   ├── 富文本编辑器 (TiptapEditor)
    │   └── 高亮列表 (HighlightList)
    └── 选中工具栏 (SelectionToolbar)
```

### 布局

- 阅读页采用三栏布局，左右面板可折叠
- 左侧目录面板：宽度约 240px，可折叠
- 中间阅读区域：自适应剩余宽度
- 右侧笔记面板：宽度约 360px，可折叠
- 面板之间支持拖拽调整宽度

### 数据流

```
用户选中文字
  → SelectionToolbar 显示
  → 用户点击"高亮"或"添加到笔记"
  → 写入 IndexedDB (highlights 表)
  → 同时插入引用块到笔记编辑器
  → 笔记自动保存到 IndexedDB (notes 表)
```

## 不做的事情（V1 scope 外）

- 云同步 / 账号系统
- 独立笔记（跨书引用）
- AI 功能（摘要、问答）
- MOBI/AZW3 格式支持
- 移动端适配
- 书内搜索
- 笔记间双向链接

## 后续规划

- **V1.1**：TXT 支持、书内搜索、笔记导出为 Markdown
- **V2**：独立笔记（跨书引用）、AI 摘要与问答、间隔复习
- **V3**：云同步、浏览器扩展（微信读书等平台笔记捕获）
